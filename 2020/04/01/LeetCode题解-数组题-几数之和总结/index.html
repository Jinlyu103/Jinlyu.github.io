      1. 三数之和

        题目描述：给你一个包含 n 个整数的数组 nums，判断 nums 中是否存在三个元素 a，b，c ，使得 a + b + c = 0 ？请你找出所有满足条件且不重复的三元组。

        注意：答案中不可以包含重复的三元组。

        示例1：

        这道题的解决思路与两数之和有些不同，如果用哈希表-登记册的方式，你需要先找到一个人作为自己的伙伴，这样做的时间复杂度为O(n^2)，显然不能这么简单粗暴。那么怎么做才能降低时间复杂度呢？

        可以采用 排序+双指针的方式

        先对数组nums进行排序，排序结果为元素从小到大排列。在求解的时候，需要注意的一点是重复解要去除

        对于第i个元素，如果i大于0，并且nums[i] == nums[i-1]，那么跳过此次循环，进入下一次循环。（去除重复解）

        设置一个左指针L = i+1，和一个右指针R= n-1

        循环条件为 L<R

        判断nums[i] + nums[L] + nums[R] 是否为0，如果为0 的话，将这三个数加入解数组中；

        如果大于0，说明右值过大，右指针需要左移

        如果小于0，说明左值过小，左指针需要右移

          1. 四数之和

        题目描述：给定一个包含 n 个整数的数组 nums 和一个目标值 target，判断 nums 中是否存在四个元素 a，b，c 和 d ，使得 a + b + c + d 的值与 target 相等？找出所有满足条件且不重复的四元组。

        注意：

        答案中不可以包含重复的四元组。

        方法和三数之和类似，也是采用排序+双指针，唯一的区别是在三数之和的基础上增加一层嵌套循环。

        对原始数组进行排序，i<j<L<R，其中L和R为定义的双指针，i和j为循环变量，i为外层循环 范围为[0,n-2) ，j为内层循环，范围为 [i+1, n-1)。
